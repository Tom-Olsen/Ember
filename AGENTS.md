# Ember Repo Instructions
This file contains repo-wide instructions for coding agents working in this repository.
Add new rules here as the project evolves.

## Editing Rules
- Do not add a trailing newline at end-of-file unless explicitly requested.
- Preserve the existing coding style of the touched file unless a broader cleanup is requested.
- Prefer ASCII in comments and source files unless non-ASCII is genuinely required.

## Review Priorities
- Prioritize logical flaws and outright bugs over style issues.
- Treat correctness and behavior as higher priority than cosmetic cleanup.
- Flag misleading comments, inconsistent naming, and structural drift.

## Communication
- Keep final responses concise and high signal.
- When you find further ideas that would make sense for me to add to this document, let me know. This includes more strict coding guidelines extracted from my existing code.
